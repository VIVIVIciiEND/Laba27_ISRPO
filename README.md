### Отыеты на вопросы 
1. Minimal API vs MVC: Minimal API - это упрощённый способ создания API без контроллеров и лишней структуры, а MVC использует контроллеры, модели и представления и подходит для больших проектов.
2. Порядок MapGet и Use: Если MapGet(...) поставить до Use(...), то middleware может не выполниться для этого запроса, потому что порядок обработки запросов важен.
3. PascalCase → camelCase в JSON: ASP.NET Core делает это по умолчанию для совместимости с JavaScript, где принято использовать camelCase.
4. HTTP коды: 200 - запрос выполнен успешно, 401 - нет авторизации, 404 - ресурс не найден.
5. dotnet run vs dotnet watch run: dotnet run просто запускает приложение один раз, а dotnet watch run автоматически перезапускает его при изменениях кода.
